# 常見問題

<details>
  <summary>Next.js 已可被用於生產了嗎？</summary>
  <p>是的！Next.js 被世界許多知名網站所使用。請參閱
  <a href="https://nextjs.org/showcase">展示</a> 來了解更多資訊。</p>
</details>

<details>
  <summary>我該如何在 Next.js 中取得資料？</summary>
  Next.js 根據您的用例提供了多種方法。您可以使用：
  <ul>
    <li> 客戶端的資料取得：在您的 React 元件中使用 <a href="/docs/basic-features/data-fetching/client-side.md#client-side-data-fetching-with-useeffect">useEffect</a> 或是 <a href="/docs/basic-features/data-fetching/client-side.md#client-side-data-fetching-with-swr">SWR</a> 來取得資料</li>
    <li> 使用 <a href="/docs/basic-features/data-fetching/get-server-side-props.md">getServerSideProps</a> 進行伺服器端渲染</li>
    <li> 使用 <a href="/docs/basic-features/data-fetching/get-static-props.md">getStaticProps</a> 生成靜態網站</li>
    <li> 通過 <a href="/docs/basic-features/data-fetching/incremental-static-regeneration.md">將 `revalidate` prop 加入到 getStaticProps</a> 來增量靜態再生</li>
  </ul>
  欲了解更多有關資料取得的資訊，請參閱我們的 <a href="/docs/basic-features/data-fetching/overview.md">資料取得文件</a>。
</details>

<details>
  <summary>為什麼 Next.js 有自己的路由器？</summary>
  Next.js 包含一個內建路由器，原因如下：
  <ul>
    <li>它使用基於檔案系統的路由器來減化配置</li>
    <li>它支援淺層路由，允許您在不執行資料取得方法的情況下更改 URL（改變搜尋參數）</li>
    <li>路由總是可以延遲加載</li>
  </ul>
    如果您正在從 React 路由器遷移至 Next.js 路由器，請參閱 <a href="/docs/migrating/from-react-router.md">遷移文件</a>。
</details>

<details>
  <summary>我可以將內容交付網路（CDN）用於靜態資產嗎？</summary>
  <p>可以！當您將您的 Next.js 應用程式部屬到 <a href="https://vercel.com">Vercel</a>，邊緣網路會自動偵測和服務您的靜態資產。如果您自行託管 Next.js，您可以在 <a href="/docs/api-reference/next.config.js/cdn-support-with-asset-prefix.md">此處</a> 學習如何手動配置資產前綴。</p>
</details>

<details>
  <summary>如何更改內部 webpack 配置？</summary>
  <p>在大多數情況下，您不必手動配置 webpack，因為 Next.js 會自動配置它。對於進階的情況，如果您需要更進一步的控制，可以參考 <a href="/docs/api-reference/next.config.js/custom-webpack-config.md">自定 webpack 配置文件</a>。</p>
</details>

<details>
  <summary>是什麼造就了 Next.js 的誕生？</summary>
  <p>我們著手完成的許多目標都列在 Guillermo Rauch的 <a href="https://rauchg.com/2014/7-principles-of-rich-web-applications">7 個多樣化網際網路應用程式的準則</a> 中。</p>

  <p>PHP 的易用性是一個很大的啟發。我們認為 Next.js 更適合取代許多使用 PHP 輸出 HTML 的情況。</p>

  <p>與 PHP 不同，我們受益於 ES6 模組系統，每個頁面都會輸出一個元件或函式，可以輕鬆導入以進行惰性求值或測試。</p>

  <p>當我們正在研究非大量步驟的伺服器端渲染 React 選項時，我們發現了 <a href="https://github.com/facebookarchive/react-page">react-page</a>（現已被棄用），這是由 React 的創作者 Jordan Walke 提供的與 Next.js 類似的方法。</p>
</details>

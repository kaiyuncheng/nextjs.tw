# 快速開始

歡迎來到 Next.js 文件!

如果你是 Next.js 的新手 ，我們建議你從 [學習課程](https://nextjs.org/learn/basics/create-nextjs-app) 開始。

這個帶有小測驗的互動式課程將會全面地帶領你去認識如何使用 Next.js。

如果你有任何關於 Next.js 的問題，歡迎隨時在我們的社群 [GitHub Discussions](https://github.com/vercel/next.js/discussions) 上提問。 

#### 系統環境需求

- [Node.js 12.22.0](https://nodejs.org/) 或更高版本
- MacOS, Windows (包括 WSL) 和 Linux 都可以支援

## 自動設置

我們建議使用 `create-next-app` 來創建一個新的 Next.js 應用程式，它會自動幫你設定好一切。要創建一個專案，請執行：

```bash
npx create-next-app@latest
# or
yarn create next-app
# or
pnpm create next-app
```

如果你想要使用 TypeScript 來開發，你可以在指令後方加上 `--typescript` 。

```bash
npx create-next-app@latest --typescript
# or
yarn create next-app --typescript
# or
pnpm create next-app --typescript
```

安裝完成後：

- 執行 `npm run dev` 或 `yarn dev` 或 `pnpm dev` 來開啟開發伺服器，位址為 `http://localhost:3000`。
- 到 `http://localhost:3000` 去查看你的應用程式。
- 編輯 `pages/index.js` 檔案，你可以在瀏覽器上看到更新的結果。

關於使用 `create-next-app` 的更多資訊，你可以查看 [`create-next-app` 文件](/docs/api-reference/create-next-app.md)。

## 手動設置

在你的專案中安裝 `next` 、 `react` 和 `react-dom`：

```bash
npm install next react react-dom
# or
yarn add next react react-dom
# or
pnpm add next react react-dom
```

打開 `package.json` 並且新增下列 `scripts` 指令碼：

```json
"scripts": {
  "dev": "next dev",
  "build": "next build",
  "start": "next start",
  "lint": "next lint"
}
```

這些指令碼對應著應用程式中不同的開發階段：

- `dev` - 執行 [`next dev`](/docs/api-reference/cli.md#development) 來啟動開發模式
- `build` - 執行 [`next build`](/docs/api-reference/cli.md#build) 來建立用於正式環境的應用程式
- `start` - 執行 [`next start`](/docs/api-reference/cli.md#production) 來啟動正式環境伺服器
- `lint` - 執行 [`next lint`](/docs/api-reference/cli.md#lint) 來設定 Next.js 內建的 ESLint 配置

在你的應用程式根目錄中新增兩個資料夾 `pages` 和 `public`：

- `pages` - 路由的路徑對應著他們的檔案名稱。 例如： `pages/about.js` 對應到 `/about`
- `public` - 存放靜態的檔案。例如： 圖片、字型等。 在 public 目錄底下的檔案可對應到專案的 base URL (`/`) 下。 

Next.js 是圍繞著 [頁面(pages)](/docs/basic-features/pages.md) 這個概念而建構出來的。 一個頁面（ Page ）就是從 `pages` 目錄資料夾底下的 `.js` 、 `.jsx` 、 `.ts` 或 `.tsx` 檔案導出的 [React 元件](https://reactjs.org/docs/components-and-props.html)。 甚至你可以在檔案名稱增加 [動態路由](/docs/routing/dynamic-routes) 的參數。

在你的專案 `pages` 資料夾下新增 `index.js` 檔案來開始。這是一個使用者訪問你的應用程式的首頁。

在 `pages/index.js` 中新增以下內容：

```jsx
function HomePage() {
  return <div>Welcome to Next.js!</div>
}

export default HomePage
```

完成設定後：

- 執行 `npm run dev` 或 `yarn dev` 或 `pnpm dev` 來開啟開發伺服器，位址為 `http://localhost:3000`。
- 到 `http://localhost:3000` 去查看你的應用程式。
- 編輯 `pages/index.js` 文件，你可以在瀏覽器上看到更新的結果。

到目前為止，我們學到了：

- 自動編譯 和 [打包 （ bundling ）](/docs/advanced-features/compiler.md) 
- [React 快速重新整理](https://nextjs.org/blog/next-9-4#fast-refresh)
- 頁面[`pages/`](/docs/basic-features/pages.md)的 [靜態生成和伺服器渲染](/docs/basic-features/data-fetching/overview.md)
- [靜態檔案](/docs/basic-features/static-file-serving.md) `public/` 對應到專案的base URL(`/`)下 

此外，任何 Next.js 應用程式從一開始就可以投入到正式環境使用。詳情請閱讀更多我們的[部署文件](/docs/deployment.md)。
